# SWRD 自进化工程级 Skill 创造器 — 初始创建经验

> Source: 本次 brainstorming + writing-plans 会话
> Collected: 2026-05-23
> Published: 2026-05-23

## 背景

基于对 5 个项目的深度调研（SkillFather、XSkill、darwin-skill、Karpathy autoresearch、Claude Autoresearch），融合各自优点，创建 SWRD 自进化工程级 Skill 创造器。

## 设计决策

### 1. 状态机设计
- SkillFather 的状态机（idle → intent_extraction → ... → success）是完备的工程化基础
- 增加了 revisit 回退路径（从任意状态回退到前一个状态重新尝试）
- 增加了 auto_optimization_loop 自进化闭环
- 最终状态机 17 个状态，覆盖创建 + 优化全流程

### 2. 评分体系
- 参考 darwin-skill 的 8 维度评分，但做了调整：
  - 3 子 Agent 投票取中位数（替代单 Agent 评分）
  - 动态阈值（低分段宽松、高分段严格）
  - 干跑验证降级方案（无法跑子 Agent 时使用）

### 3. 全日志系统
- YAML 主日志（人类可读）+ JSONL 训练集（机器可解析）
- 记录粒度：每次调用、每次 Eval、每次优化迭代、每次回退
- 日志结构包含 session_id、agent_id、分数、决策等完整信息

### 4. 模块化约束
- 每个模块 ≤ 3 文件、≤ 300 行
- 创建时通过子 Agent 审查
- 高内聚低耦合从设计初期约束

### 5. 版本号策略
- 三段式，0.1.0 起始
- 棘轮前进升最小版本
- 禁止回退版本号
- 唯一位置 SKILL.md frontmatter

### 6. 联动工程适配
- 通用适配层：git 仓库 / 非 git 目录 / 自动 init / 全量备份
- 每次实验前快照，实验后恢复

## 经验教训

### 教训 1：模板文件必须先于 SKILL.md 中的引用存在
创建时 SKILL.md 中写了"从模板生成"，但 assets/skill-template/ 为空。导致 D5 指令具体性初始只有 5 分。
**解决方案**：第一轮优化立即填充了 7 个模板文件和 2 个脚本。

### 教训 2：边界条件容易被忽略
初始版本只关注了正常流程，忽略了模板缺失、git 失败、子 Agent 超时等边界情况。
**解决方案**：第二轮优化补充了 8 类边界条件的处理逻辑。

### 教训 3：工作流步骤缺少输入/输出标注
初始版本的工作流步骤只有操作描述，没有明确标注每个阶段的输入、输出、退出条件。
**解决方案**：第三轮优化为 11 个阶段全部补充了结构化标注。

### 教训 4：实测表现评分标准需要量化
初始版本的 D8 评分标准只有模糊描述（"输出质量符合 skill 宣称的能力"），无法客观评分。
**解决方案**：细化为 1-10 分对照表 + 三维度评估法 + 干跑验证降级方案。
