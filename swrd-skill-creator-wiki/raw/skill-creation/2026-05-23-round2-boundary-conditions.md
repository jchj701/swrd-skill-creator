# 第二轮棘轮进化 — 补充边界条件

> Source: 本次会话
> Collected: 2026-05-23
> Published: 2026-05-23

## 问题

初始版本只关注了正常流程，忽略了大量边界情况。执行约束中只有 9 条禁止规则，缺少对异常场景的处理指导。

## 改进内容

### 补充的边界条件（8 类）
1. 模板文件缺失 → 使用内嵌默认模板，不中断流程
2. git init 失败 → 检查是否已是 git 仓库，或提示用户手动初始化
3. git commit 失败 → 尝试环境变量，提示用户配置
4. 子 Agent 超时（1个）→ 使用剩余 2 个结果
5. 子 Agent 超时（≥2个）→ 降级为主 Agent 自检，标记 degraded_mode
6. 联动工程路径不存在 → 提示用户，询问创建或更换
7. 联动工程路径无权限 → 降级为只读模式
8. 用户未指定联动工程 → 使用临时目录，测试后自动清理
9. 用户中途取消 → 停止操作、清理临时文件、恢复联动工程
10. 磁盘空间不足 → 写入前检查
11. 文件写入冲突 → 不直接覆盖，提示用户
12. 版本号解析失败 → 校验后使用

### 执行约束补充
新增 8 条禁止规则和 6 条必须规则，覆盖上述边界场景。

## 效果
- D3 边界条件覆盖：6 → 9
- 总分：68 → 79

## 经验
- 边界条件应该在设计阶段就系统性地考虑，而不是等到评分低再补充
- 一个系统性方法：对每个外部操作（git、文件写入、子 Agent 调用）问"如果失败了怎么办"
- 降级方案比直接报错更好：能继续就继续，不能继续再停止
