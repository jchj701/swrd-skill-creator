# 用户提出的关键约束

> Sources: 本次 brainstorming 会话中的用户输入, 2026-05-23
> Raw: [用户约束原始素材](../../raw/design-philosophy/2026-05-23-user-constraints.md)

## 概述

以下约束是用户在本会话中明确提出的，构成了 SWRD 区别于其他 Skill 的核心特征。

## 约束清单

| 约束 | 用户原始表述 | 设计实现 |
|------|-------------|----------|
| 版本号纪律 | "三段式版本号，棘轮前进升最小版本，禁止回退" | SKILL.md 唯一版本号，CHANGELOG.md 记录变更 |
| 全日志 | "越细越好，跨任务追加，用于二次训练" | YAML 主日志 + JSONL 训练集 |
| 模块化 | "从创建时就约束，高内聚低耦合" | ≤3 文件/模块，子 Agent 审查 |
| 离线可用 | "可能离线环境，无法联网" | karpathy-llm-wiki 零依赖 |
| 联动工程通用 | "不一定是代码或 git 仓库" | 通用适配层 |
| 评分独立 | "需要独立的专家自动审核" | 3 子 Agent 投票 |
| 棘轮粒度 | "每一轮都可测可回溯" | git tag + 独立评分 |

## 经验

用户的原始表述应该保留在 raw/ 中，不要过度加工。用户的措辞往往比 Agent 的理解更准确地反映了真实需求。
