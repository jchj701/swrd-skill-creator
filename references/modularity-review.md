# 模块化审查规则

## 审查规则

| 编号 | 规则 | 严重级别 | 自动修复 |
|------|------|----------|----------|
| M1 | 每个模块不超过 3 个文件 | error | 否 |
| M2 | 每个文件不超过 300 行 | error | 否 |
| M3 | 模块间无循环依赖 | error | 否 |
| M4 | 每个模块职责单一 | warning | 否 |
| M5 | 模块间接口在 skill.yaml 中明确定义 | warning | 否 |
| M6 | 禁止跨模块引用内部文件 | error | 否 |

## 审查流程

1. spawn 3 个子 Agent
2. 每个子 Agent 独立审查所有规则
3. 汇总结果，取多数决
4. ≥2 子 Agent 判定 error → 审查不通过
5. 审查不通过 → 回退到 Skill 生成阶段重新拆分

## 模块化设计原则

- 每个模块 = 一个逻辑单元 = 1-3 个文件
- 模块间通过 skill.yaml 中的 workflow.steps 定义依赖
- 禁止跨模块引用（一个模块不能直接读取另一个模块的内部文件）
- 每个模块职责单一（一个模块只做一件事）
- 模块间接口清晰定义
