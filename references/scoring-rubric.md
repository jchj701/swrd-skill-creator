# 8 维度评分细则

## 评分总览

总分 100。结构维度靠静态分析（60 分），效果维度必须实测（40 分）。

## 结构维度（60 分）

### D1: Frontmatter 质量（权重 8）
- name 符合规范（小写字母、数字、连字符）
- description 包含做什么 + 何时用 + 触发词
- description ≤ 1024 字符
- version 字段存在且格式正确

### D2: 工作流清晰度（权重 15）
- 步骤明确可执行
- 有序号
- 每步有明确输入/输出
- 有状态机定义

### D3: 边界条件覆盖（权重 10）
- 处理异常情况
- 有 fallback 路径
- 有错误恢复机制

### D4: 检查点设计（权重 7）
- 关键决策前有用户确认
- 防止自主失控
- 人在回路机制

### D5: 指令具体性（权重 15）
- 不模糊
- 有具体参数/格式/示例
- 可直接执行

### D6: 资源整合度（权重 5）
- references/scripts/assets 引用正确
- 路径可达
- 依赖声明完整

## 效果维度（40 分）

### D7: 架构合理性（权重 15）
- 结构层次清晰
- 不冗余不遗漏
- 模块化设计合理
- 高内聚低耦合

### D8: 实测表现（权重 25）
- 用测试 prompt 跑一遍
- 输出质量符合 skill 宣称的能力
- 相比不带 skill 的 baseline 有提升

## 评分规则
- 每个维度打 1-10 分，乘以权重得到该维度得分
- 总分 = Σ(维度分 × 权重) / 10，满分 100
- 3 个子 Agent 独立评分，取中位数
- 动态阈值：总分 < 60 时 Δ ≥ 1 算提升；60-80 时 Δ ≥ 2 算提升；≥ 80 时 Δ ≥ 1 算提升
