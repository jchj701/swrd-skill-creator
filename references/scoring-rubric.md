# 8 维度评分细则

## 评分总览

总分 100。结构维度靠静态分析（60 分），效果维度必须实测（40 分）。

## 结构维度（60 分）

### D1: Frontmatter 质量（权重 8）
- name 符合规范（小写字母、数字、连字符）
- description 包含做什么 + 何时用 + 触发词
- description ≤ 1024 字符
- version 字段存在且格式正确

### D2: 工作流清晰度（权重 15）
- 步骤明确可执行
- 有序号
- 每步有明确输入/输出
- 有状态机定义

### D3: 边界条件覆盖（权重 10）
- 处理异常情况
- 有 fallback 路径
- 有错误恢复机制

### D4: 检查点设计（权重 7）
- 关键决策前有用户确认
- 防止自主失控
- 人在回路机制

### D5: 指令具体性（权重 15）
- 不模糊
- 有具体参数/格式/示例
- 可直接执行

### D6: 资源整合度（权重 5）
- references/scripts/assets 引用正确
- 路径可达
- 依赖声明完整

## 效果维度（40 分）

### D7: 架构合理性（权重 15）
- 结构层次清晰
- 不冗余不遗漏
- 模块化设计合理
- 高内聚低耦合

### D8: 实测表现（权重 25）

**评分方法**：为 Skill 设计 2-3 个典型测试 prompt，用子 Agent 分别执行带 Skill 和不带 Skill 的版本，对比输出质量。

**评分标准（1-10 分）**：

| 分数 | 标准 |
|------|------|
| 1-2 | 带 Skill 的输出质量低于不带 Skill 的 baseline，Skill 起反作用 |
| 3-4 | 带 Skill 的输出与 baseline 持平，Skill 无明显价值 |
| 5-6 | 带 Skill 的输出略优于 baseline，但提升不明显或仅部分场景有效 |
| 7-8 | 带 Skill 的输出明显优于 baseline，覆盖大部分场景，输出结构清晰 |
| 9-10 | 带 Skill 的输出显著优于 baseline，覆盖所有场景，输出精准且无副作用 |

**评估维度**：
1. **意图完成度**：输出是否完成了用户意图？（0-3 分）
2. **质量提升度**：相比不带 Skill 的 baseline，质量提升是否明显？（0-4 分）
3. **副作用控制**：Skill 是否引入了负面影响（过度冗余、跑偏、格式奇怪）？（0-3 分）

**降级方案**：
- 如果无法跑子 Agent（时间/资源限制），退化为「干跑验证」：读完 Skill 后模拟一个典型 prompt 的执行思路，判断流程是否合理
- 干跑验证在 results.tsv 中标注 `dry_run`
- 干跑验证最高得 6 分（无法验证实际效果）

## 评分规则
- 每个维度打 1-10 分，乘以权重得到该维度得分
- 总分 = Σ(维度分 × 权重) / 10，满分 100
- 3 个子 Agent 独立评分，取中位数
- 动态阈值：总分 < 60 时 Δ ≥ 1 算提升；60-80 时 Δ ≥ 2 算提升；≥ 80 时 Δ ≥ 1 算提升
